**Holded** — это глючный инструмент фиксации фактов. Используется как промежуточное хранилище, мы переезжаем с него в Odoo.

**Что брать с Holded:**
1. **Бумажные PDF от поставщика** — единственная правда о цене/SKU/qty/concepto. Скачивать через URL `https://app.holded.com/box/file?p=purchaseshipments/<mongoId>/deliverynote<fileId>.pdf`. Эти PDF потом подгружать в `ir.attachment` каждого pedido в Odoo.
2. **Иногда нештатные консультации** — но лучше спросить пользователя.

**Что НЕ брать с Holded:**
- Holded API products array (price, units, sku, name) — это **результат работы сотрудников**, могли ошибиться при ручном вводе бумажного альбарана. Не источник истины.
- Holded recount qty в Odoo lines — это **физический пересчёт сотрудником**, может расходиться с бумагой; полезно как «что реально приехало», но цена/SKU/concepto должны браться с paper.

**Why:** owner явно подтвердил — «Holded это просто глючный инструмент фиксации фактов мы с него переезжам на odoo и попутно правим факты какие можем восстановить корректно. что не сможем — поправим инвентаризацией. бумага от поставщика всё же довольно корректный документ».

**Алгоритм закрытия pedido — сверка двух источников:**
- **PDF от поставщика** = истина (price, Verdnatura Ref, concepto, ordered qty)
- **Odoo pedido lines** = мнение бухгалтера (какую Odoo карту он сматчил + сколько реально пересчитал)
- Закрытие = сверить оба, найти совпадения и расхождения, записать paper-truth по price/SKU/concepto + сохранить Odoo recount qty (или paper qty если есть подозрение бухгалтер ошибся)

**How to apply:**
- Reconcile pedido — **только при наличии paper PDF**. Если PDF нет — не закрывать. Pedido в state=draft до появления PDF.
- Холодные данные (price/sku из API) использовать как **подсказки** для рекомендаций владельцу, не как факты для записи в Odoo.
- Я уже один раз нарушил это правило (pedido 37724 / 12492439, 29-04-2026) — закрыл на Holded API данных без paper. Owner назвал «большая ошибка». Phase A откачена.
