# 11. CRM и клиенты

Статус: 🔴 **CONCEPT** — не реализовано, нет брифа. Этот файл — целеполагание и scope.

---

## Зачем CRM

Цветочный бизнес — **emotional, high-repeat, relationship-driven**:
- Клиенты возвращаются на **личные даты** (дни рождения, годовщины, 8 марта подарок маме и т.д.)
- Цена покупки выше средней, маржа выше средней → важно удержание
- Сарафанное радио = сильный канал → лояльность через персональное отношение

**Цель CRM:** превратить каждую продажу в **начало long-term relationship**, а не одноразовую транзакцию.

## Scope функциональности

### 11.1. Клиентская карточка

**Базовые данные:**
- Имя, телефон, email, адрес (для доставки)
- Канал привлечения (offline / Flowwow / Glovo / Instagram / сайт / рекомендация)
- Дата первой покупки
- История заказов (полная)

**Важные персональные даты:**
- День рождения самого клиента
- Дни рождения / годовщины близких (жена, мама, дочь и т.д.)
- Годовщины отношений
- Прочие «он дарит цветы X раз в году по таким-то поводам»

**Система должна напоминать:**
- За 7-10 дней до значимой даты — alert сотруднику или клиенту
- Предложить повторить «удачный» букет прошлого года
- Предложить подписку «каждый месяц жене букет»

### 11.2. Лояльность и скидки

**Персональные скидки:**
- % скидка постоянному клиенту
- Бонусные баллы (спасибо-балл за каждый евро)
- Карточка лояльности (виртуальная — QR / номер телефона)

**Сегменты:**
- VIP (топ-20% по обороту)
- Regular (2+ покупки за квартал)
- Dormant (не покупал 3+ месяца) → риск потери
- New (первая-вторая покупка) → нужна wow-экспа

**Промо-акции:**
- Happy hour
- Сезонные скидки
- Для конкретного сегмента (email dormant-клиентам)

### 11.3. Рассылки (marketing outreach)

**Каналы:**
- **Email** — новости, сезонные предложения, дайджесты
- **WhatsApp** — персональные напоминания о датах, статус заказа
- **SMS** — срочные алерты (готов заказ, курьер выехал)
- **Telegram** — если клиент активен в Telegram
- **Instagram** — визуальные промо через DM

**Принципы:**
- **Opt-in по каналам** (клиент выбирает, где его беспокоить)
- Частота — умеренная (максимум 1-2 рассылки в месяц на клиента)
- Персональный контент > массовый

### 11.4. Привязка клиента к продаже

**В POS — быстрый лукап:**
- По номеру телефона (самый частый)
- По имени (если клиент узнаёт флорист)
- По QR-карте лояльности

**Не замедлять работу флориста:**
- Если клиент новый → быстрое создание карточки (только телефон, остальное потом)
- Если клиент анонимный → продажа без привязки (случается)
- Кнопка «добавить этого клиента сейчас» на чеке уже после продажи

**Прикрепление важных данных:**
- Если клиент заплатил за доставку → адрес доставки в карточку
- Если попросил карточку к букету → текст карточки в историю
- Если сделал фото с букетом → привязать (опционально)

### 11.5. Обработка входящих запросов

**Каналы входа:**
- Звонок в магазин
- WhatsApp бизнес
- Instagram Direct
- Flowwow / Glovo (marketplace, но после заказа — клиент наш)
- Заявка с сайта

**Единая воронка:**
- Все входящие → одно место в Odoo (`crm.lead` / `mail.channel`)
- Привязка к обрабатывающему сотруднику (для бонусов, см. [05](05_florists_logistics_accountant.md))
- Timeline общения
- Конвертация в sale.order при оформлении

## Что есть в Odoo из коробки

**Штатные модули:**
- `crm.lead` — лиды и opportunities
- `res.partner` — unified customer entity (и для продаж, и для закупок)
- `account.move` — история покупок привязана к `partner_id`
- `loyalty.program` — программы лояльности (скидки, баллы)
- `mass.mailing` — email-рассылки
- `sms` (платный модуль) — SMS
- `website_sale` — eCommerce (если включим свой сайт, [10 шаг 13](10_vision_and_roadmap.md))

**Чего нет из коробки:**
- WhatsApp интеграция (нужен сторонний модуль или Make.com)
- Instagram DM (через сторонний API)
- Кастомные даты-напоминалки (можно через `mail.activity` + scheduler)
- Сегментация по кастомным правилам (частично штатная, частично custom)

## Roadmap

**MVP (с запуска 20 апреля):**
- Базовая карточка клиента в POS (создаётся при продаже по телефону)
- История покупок привязана через `partner_id`
- Без активных рассылок, без сегментации

**Этап 2 (после 2-3 месяцев данных):**
- Сегментация (VIP / Regular / Dormant / New)
- Первые email-рассылки через `mass.mailing`
- Simple loyalty program (баллы / процент)

**Этап 3 (с ростом канала online):**
- WhatsApp интеграция
- Напоминания о значимых датах
- Первые автоматические промо для Dormant

**Этап 4 (долгосрочное):**
- Instagram DM интеграция
- Подписочные модели
- Advanced predictive analytics (прогноз follow-up покупки)
